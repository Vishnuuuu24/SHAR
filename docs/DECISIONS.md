# Research decisions — SHAR

> v3.0 · 2026-07-20 · Research and governance decisions; effective scope remains synchronized in higher-authority documents.

| ID | Decision | Reason and consequence |
|---|---|---|
| D-01 | DR-4 is resolved: the local Kaggle images are 64×64. | Stage 2 cannot make credible small-action localization claims from these images alone. Use original-resolution media or an independently annotated localization dataset. |
| D-02 | Call Kaggle UCF training “video-label-inherited weak/noisy frame supervision.” | The source UCF training split has video-level labels; untrimmed anomalous videos contain normal frames. |
| D-03 | Main UCF test is official-interval-derived and event-only. | Score frames inside official anomaly intervals plus frames from Normal test videos; exclude out-of-interval frames from anomalous videos from the headline multiclass score. Keep the inherited-label full-directory score only as a noisy-proxy sensitivity analysis. Do not call the interval labels clean or exhaustive. |
| D-04 | Eliminate manual annotation from the core plan. | Use existing real annotations. Pseudo-labels may propose ROIs or auxiliary training targets but may never be evaluation ground truth. |
| D-05 | Core Stage 2 is stock-detector ROI preprocessing, evaluated by downstream UCF macro-F1. | Avenue anomaly masks are not instance masks, and pseudo-label-versus-pseudo-label mAP is circular. UCF-Crime2Local/COCO/AVA metrics are optional independent checks or prerequisites for revived custom-detector work. |
| D-06 | AVA v2.2 is the preferred movie dataset, separately evaluated. | It has licensed movie clips, person boxes, atomic action labels, official splits, and Frame-mAP tooling. Its ontology and domain are not equivalent to UCF’s 14 composite labels. |
| D-07 | XD-Violence and MSAD are external generalization candidates; RareAnom remains valuable but optional. | All solve different label/task regimes and therefore receive separate tables, mappings, and metrics. No label merging into the UCF headline. |
| D-08 | Remove the absolute “no 14-class work exists” claim. | Multiple published works already study 14-category/video or frame-derived UCF-Crime classification. The defensible contribution is the controlled protocol, provenance, noise study, and independent ablations. |
| D-09 | Defer custom MSDAM detector work. If revived, keep stock YOLO11 P5 C2PSA and test standalone residual MSDAM after P3 and/or P4. | Official YOLO11 has C2PSA only after SPPF at P5. DilateFormer supports shallow MSDA in its own transformer, not a proven P3/P4 mapping. Independent detection labels and a new decision are required. |
| D-10 | Remove Gaussian ASFFH from core scope. Standard ASFF is the mandatory future control. | The proposed Gaussian equation lacked a reference, alignment, normalization, and evidence of benefit. Preserve it only as a deferred, fully specified prototype if custom detection is revived. |
| D-11 | Remove Slide Loss and its confidence-classification port from core scope. | It has no class-frequency term, local MS-YOLO reports worse Slide-only mAP, and YOLO-FaceV2 paper/code disagree (`exp(1-μ)` versus `exp(μ)`). Any revival must name and ablate variants. |
| D-12 | Rename the visual adaptation to “SEMSCNN-inspired 2-D multiscale-SE head” and feed ResNet pre-GAP spatial features. | GAP removes spatial topology; reshaping a 2048-D vector creates arbitrary adjacency. The source is EEG channel×time with temporal kernels, so visual choices are explicit adaptations. |
| D-13 | Use a stable cryptographic digest for per-image noise seeds and experiment IDs. | Python’s built-in `hash()` is randomized between interpreter processes unless externally fixed. |
| D-14 | Keep YOLO11 as the stock ROI teacher; use an optional frozen YOLO26 like-for-like row only if labeled detector work is revived. | YOLO26 is the current official Ultralytics generation. YOLO12 uses the API but is community-driven, lacks official pretrained segmentation weights, and has documented stability/resource cautions. |
| D-15 | Primary sources govern; Reddit is a documented sanity check only. | Reddit has useful practitioner warnings but no representative consensus for paper equations, dataset truth, or novelty. Claims are never accepted because “most Reddit users agree.” |
| D-16 | Make the MacBook Pro M5 Pro/48 GB the primary quality-first execution target. | Use MPS with explicit fallback logging, full intended research settings, resumable runs, and measured pre-run time/storage/memory ranges. Calibration tests are allowed, but silent speed-driven downscaling is not. |
| D-17 | Adopt a four-part execution-control system: `PHASE_PLAN`, `PROJECT_CONTROL`, `PROJECT_STATUS`, and `TRAINING_LOG`. | Stable roadmap, operating rules, live state, and growing run history change at different rates; separating them prevents stale status and duplicated scientific authority. |
| D-18 | Permit Phase-0 validation/foundation implementation while keeping research training gated. | Phase 0 requires mapping, provenance, environment, tests, and compute calibration tooling; prohibiting all code made its own exit gate impossible. |
| D-19 | Make original-resolution UCF and UCF-Crime2Local conditional rather than core blockers. | Original media is required only for activated high-resolution ROI work; independent localization annotations are required only for mAP/custom-detector work. The core stock-ROI study uses downstream classification. |
| D-20 | Maintain one concise append-preserving training index linked to immutable append-only attempt artifacts. | Logical-experiment rows may advance through their lifecycle but are never deleted; every attempt—including failures and negative/invalid outcomes—remains reviewable without copying epoch telemetry into Markdown. |
| D-21 | Separate code-complete, run-complete, and research-complete; completion does not require a positive result. | This prevents implementation or execution from being mistaken for evidence. Evaluated claims end as supported, not supported, or inconclusive. |
| D-22 | Freeze teacher ROI provenance as `label_scope=teacher_roi` with `label_source=teacher`; keep checkpoint identity in linked provenance. | FR-8 has authority over the lower DATA_SPEC wording. This resolves the conflict without weakening the rule that teacher outputs never become evaluation ground truth. |
